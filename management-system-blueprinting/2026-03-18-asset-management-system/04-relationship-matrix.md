# 关系矩阵

## 业务面关系矩阵

| 关系类型 | 来源 | 去向 | 关系说明 |
| --- | --- | --- | --- |
| 上下游依赖 | 资产取得与建账 | 统一资产台账 | 新增取得和存量补录必须先完成建账，才能成为正式在管资产 |
| 上下游依赖 | 统一资产台账 | 资产归属与使用 | 只有正式在册且具备可用条件的资产才能进入使用流程 |
| 上下游依赖 | 资产归属与使用 | 盘点、巡检与维护 | 使用关系决定盘点对象、巡检对象和责任对象 |
| 上下游依赖 | 盘点、巡检与维护 | 处置与退出 | 差异、毁损、闲置和核查结果会触发处置流程 |
| 上下游依赖 | 处置与退出 | 价值与财务协同 | 处置结果必须先形成财务协同口径 |
| 上下游依赖 | 价值与财务协同 | 统一资产台账 | 财务协同完成后，台账才能进入终态 |
| 触发关系 | 统一资产台账 | 资产归属与使用 | 资产建账完成触发领用、分配或占用流程 |
| 触发关系 | 资产归属与使用 | 价值与财务协同 | 启用、归属变化、位置变化会触发财务协同更新 |
| 触发关系 | 资产归属与使用 | 盘点、巡检与维护 | 在用资产进入周期性盘点、巡检或维修 |
| 触发关系 | 盘点、巡检与维护 | 统一资产台账 | 正常盘点结果触发台账修正 |
| 触发关系 | 盘点、巡检与维护 | 处置与退出 | 异常结果触发待处置流程 |
| 状态传递 | 资产取得与建账 | 统一资产台账 | 草稿、待建账、在册状态由建账过程推进 |
| 状态传递 | 统一资产台账 | 资产归属与使用 | 在册状态可转为使用中、闲置中等运行状态 |
| 状态传递 | 资产归属与使用 | 盘点、巡检与维护 | 使用中资产进入盘点中、核查中、整改中 |
| 状态传递 | 盘点、巡检与维护 | 处置与退出 | 不可用、毁损、退出条件成立时转为待处置 |
| 状态传递 | 处置与退出 | 统一资产台账 | 财务确认后转为已处置或已注销 |
| 数据归属 | 统一资产台账 | 全部业务面 | 台账是资产主数据、状态、归属和价值基础信息的唯一底座 |
| 数据归属 | 资产归属与使用 | 盘点、财务协同 | 使用关系、责任人、位置和启用时间是盘点与财务的重要业务输入 |
| 数据归属 | 盘点、巡检与维护 | 台账、处置 | 盘点差异、巡检异常和整改结果回写台账或触发处置 |
| 数据归属 | 价值与财务协同 | 台账 | 财务确认结果决定终态口径和对账依据 |
| 协作关系 | 资产管理员 | 部门领用人 | 共同完成领用、退还、盘点和责任确认 |
| 协作关系 | 资产管理员 | 财务 | 共同确认原值、启用日期、处置口径 |
| 协作关系 | 部门领用人 | 财务 | 通过使用状态和交接结果间接支撑价值口径核对 |

## 关键关系结论

- 台账是唯一资产主档底座，不能被其他业务面绕过
- 使用关系是盘点和财务协同的共同上游
- 盘点结果不是终点，而是台账修正和处置决策的触发器
- 处置不是孤立动作，必须同时影响财务口径和台账终态
