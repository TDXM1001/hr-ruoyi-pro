# 生命周期与状态流转

## 核心对象

核心对象定义为“资产记录”。固定资产和不动产共用统一生命周期框架，只在子状态和专属业务规则上区分。

## 状态流转表

| 状态 | 进入条件 | 可流转到 | 触发角色/动作 |
| --- | --- | --- | --- |
| 草稿 | 发起新增取得或存量补录 | 待建账 | 资产管理员创建资产草稿 |
| 待建账 | 基础资料已收集但未完成正式建账 | 在册、待核查 | 资产管理员补齐编码、分类、归属、价值信息 |
| 待核查 | 资料不全、重复疑似或关键信息异常 | 待建账、已注销 | 资产管理员核查异常信息 |
| 在册 | 完成正式建账，形成正式资产记录 | 使用中、闲置中、盘点中、待处置 | 资产管理员确认建账完成 |
| 使用中 | 完成领用、分配、占用或启用 | 闲置中、盘点中、整改中、待处置 | 资产管理员、部门领用人执行使用确认 |
| 闲置中 | 已在册但暂未使用或暂未分配 | 使用中、待处置 | 资产管理员调整归属或状态 |
| 盘点中 | 进入盘点、巡检或专项核查 | 使用中、整改中、待处置 | 资产管理员发起盘点或巡检 |
| 整改中 | 发现故障、异常、需要维修或整改 | 使用中、待处置 | 资产管理员推动维修或整改闭环 |
| 待处置 | 满足报废、出售、转让、注销等退出条件 | 已处置、已注销、使用中 | 资产管理员发起处置；未完成可回退 |
| 已处置 | 完成处置与财务协同 | 终态 | 财务和资产管理员完成确认 |
| 已注销 | 完成注销与财务协同 | 终态 | 财务和资产管理员完成确认 |

## 状态流转说明

### 固定资产常见路径

- 草稿 -> 待建账 -> 在册 -> 使用中 -> 盘点中 -> 使用中 -> 待处置 -> 已处置
- 草稿 -> 待建账 -> 在册 -> 闲置中 -> 待处置 -> 已注销
- 草稿 -> 待建账 -> 在册 -> 使用中 -> 整改中 -> 使用中

### 不动产常见路径

- 草稿 -> 待建账 -> 在册 -> 使用中 -> 盘点中 -> 使用中
- 草稿 -> 待建账 -> 在册 -> 使用中 -> 整改中 -> 使用中
- 草稿 -> 待建账 -> 在册 -> 使用中 -> 待处置 -> 已处置
- 草稿 -> 待建账 -> 在册 -> 待核查 -> 待建账

## 统一状态设计的好处

- 资产管理员维护一套统一状态体系
- 财务看到统一状态口径，便于对账
- 固定资产和不动产可以统一统计和统一预警
- 只在子规则层区分资产类型，不在顶层状态上分裂
