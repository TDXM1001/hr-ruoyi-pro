# 若依系统全量接入设计方案

## 1. 概述 (Overview)

本项目目标是将现有的若依后端（Spring Boot, 位于 `RuoYi-Vue`）的核心「系统管理」全量对接到采用现代技术栈的 `art-design-pro`（Vue 3 + Vite + Element Plus + Tailwind CSS）前端中。本方案明确了对接的核心架构设计、基础设施改造规范及业务模块移植路径。

## 2. 核心架构与基础设施设计 (Core Infrastructure)

### 2.1 HTTP 引擎改造方案
- **路径重构**：重构 `src/utils/http`，移除原有用于模拟请求的状态判定。
- **状态码规范**：依据若依标准设计状态拦截器，处理如下关键状态码：
  - `200`：请求成功响应。
  - `401`：Token 失效或未授权，需清除本地状态并跳转登录。
  -  其他（如 `500`）：通过 `ElMessage` 全局错误提示。
- **Token 携带**：所有接口请求需在 Request Header 中注入 `Authorization: Bearer <Token>` 形式的认证信息。

### 2.2 动态路由与权限拦截机制
- **路由数据获取**：取消基于本地强写死的路由加载策略。用户登录时调用 `/getInfo` 获取「用户角色」与「权限字符」，后调用 `/getRouters` 获取「动态菜单路由」。
- **动态路由解析**：编写树形数据转换引擎，将 `/getRouters` 下发的数据（包含父子菜单层级、组件路径等）解析转化为 `vue-router` 对象实例 `RouteRecordRaw` 并使用 `router.addRoute` 挂载。
- **自定义指令系统**：基于权限字符与角色信息，实现 `v-hasPermi` 和 `v-hasRole` 全局指令，以精细控制按钮级及组件级渲染。

### 2.3 常用工具链封装 (Utils Framework)
全面移植若依特色且高频依赖的方法，作为辅助系统的底层引擎：
- `handleTree`：通用树形数据结构构造函数。
- `resetForm`：重置表单对象及相关校验状态。
- `parseTime`：基于业务的时间格式化工具。
- `download`：二进制文件或通用业务流下载封装。

## 3. 字典体系的高效复用 (Dictionary System)

字典体系是若依系统渲染和状态显示的关键。
- **Pinia 全局存储**：通过构建 `useDictStore` 中心化状态管理处理所有按需拉取和预拉取得来的数据字典对象。
- **Hook 机制**：完整复用 `useDict` 组件级钩子以简易快速地声明所依赖的类别并在组件内存取状态。
- **Tag 标签渲染**：移植 `<DictTag>` 组件并配以 Tailwind CSS 装饰类，为诸如状态、性别等字段提供颜色与文案关联映射功能。

## 4. 若依核心系统模块落地规划 (System Modules Transfer)

在完成以上基建步骤后，逐步实现若依提供的完整系统功能界面：
1. **基础组织架构**：完成「用户管理」（包含部门数配合）、「部门管理」、「岗位管理」。
2. **权限与菜单体系**：完成「角色管理」（包含数据权限绑定）、「菜单管理」。
3. **系统基础配置**：完成「字段类型/数据管理」（涵盖应用层核心枚举维护）、「参数配置」。
4. **日志与通告机制**：完成「通知公告」以及「操作日志」、「登录日志」等审计行为记录展现。

所有的业务界面开发都将采用 `Element Plus` 框架和 `Tailwind CSS` 进行重写和排版，实现美观的现代后台管理系统界面，且同时保留所有若依既有数据的兼容和通信协议。
