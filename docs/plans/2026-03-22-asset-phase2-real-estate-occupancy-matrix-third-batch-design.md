# 2026-03-22 不动产占用状态矩阵第三批设计

## 1. 背景
当前不动产占用页签已经具备：
- 当前有效占用卡片
- 状态矩阵（无有效占用 / 存在有效占用 / 已释放历史）
- 状态筛选与关键字筛选
- 发起 / 变更 / 释放 Drawer

但从资产管理员使用路径看，还存在 3 个明显断点：
- 变更完成后，当前卡片只展示“当前值”，缺少“这次变更到底改了什么”的联动摘要
- 释放完成后，空态只有“暂无有效占用”，缺少最近释放信息与下一步引导
- 轨迹筛选只有状态和关键字，没有时间维度，无法快速看最近 7/30/90 天占用变化

本批目标是继续强化占用页签的状态矩阵能力，让资产管理员在详情页内更容易理解：当前处于哪个状态、最近刚发生了什么、下一步应该做什么。

## 2. 范围与边界
### 2.1 本批包含
- 变更后摘要卡片联动强化
- 释放后空态引导
- 轨迹时间维度筛选
- 对应前端单测与浏览器点测

### 2.2 本批不包含
- 不新增后端接口
- 不新增 SQL
- 不新增一级/二级路由
- 不修改处置、审批、整改相关模块

## 3. 方案选择
### 方案 A：仅在现有卡片上补文案
优点：改动最小。
缺点：提示信息零散，难形成状态认知。

### 方案 B：在占用页签内补“摘要联动 + 空态引导 + 时间筛选”（推荐）
优点：实现集中在占用页签，用户路径不跳转，返工最少。
缺点：需要补充更多派生视图和筛选逻辑。

### 方案 C：拆独立占用轨迹页
优点：后续扩展空间更大。
缺点：当前阶段过重，违背“最小闭环增强”。

结论：采用方案 B。

## 4. 详细设计
### 4.1 变更后摘要卡片联动强化
当存在有效占用时，在“当前有效占用”卡片内新增两块内容：

1. 主档联动摘要
- 展示主档当前使用部门
- 展示主档当前责任人
- 展示主档当前使用位置
- 若主档与当前有效占用一致，则展示“主档已同步”成功提示
- 若出现不一致，则展示“主档待校正”警示提示

2. 最近一次变更摘要
- 当存在上一条已释放占用记录时，展示：
  - 上一条占用单号
  - 上一条部门 / 责任人 / 位置
  - 当前条部门 / 责任人 / 位置
  - 本次生效日期
  - 本次原因说明
- 当不存在上一条历史记录时，展示“当前为首条有效占用”提示

### 4.2 释放后空态引导
当不存在有效占用时，空态卡片从“静态提示”升级为“最近释放 + 下一步动作”：
- 最近释放单号
- 最近释放日期
- 最近释放原因
- 引导动作：
  - 发起占用
  - 查看已释放轨迹（自动切到 `RELEASED` 状态筛选并滚动到轨迹区）
- 若没有任何历史记录，仍保留纯空态文案

### 4.3 轨迹时间维度筛选
在现有状态/关键字筛选基础上，再补：
- 全部时间
- 近 7 天
- 近 30 天
- 近 90 天

时间判定规则：
- 已释放记录优先使用 `endDate`
- 有效占用记录使用 `startDate`

筛选逻辑最终为三层叠加：
- 状态筛选
- 时间筛选
- 关键字筛选

### 4.4 用户体验要求
- 时间筛选和状态筛选保持并列，不额外增加跳页
- 空态按钮和引导文案要放在首屏可见区域
- 摘要卡片与轨迹区仍保持整页连续滚动，不引入内部小滚动容器

## 5. 测试设计
### 5.1 单测
新增/补充以下断言：
- 当前有效占用场景能看到“主档联动摘要”
- 存在已释放上一条记录时能看到“最近一次变更摘要”
- 无有效占用但存在释放历史时，能看到最近释放信息与“查看已释放轨迹”按钮
- 点击“查看已释放轨迹”后，只显示已释放记录
- 时间筛选切到近 30 天后，较旧记录被排除

### 5.2 点测
浏览器点测确认：
- 占用页签首屏出现摘要强化内容
- 释放后空态能直接引导到已释放轨迹
- 时间筛选按钮真实生效
