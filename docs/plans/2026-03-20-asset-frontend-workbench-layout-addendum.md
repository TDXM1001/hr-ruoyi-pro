# 2026-03-20 资产模块前端工作台骨架补充

## 1. 目的

本补充文档用于细化资产模块列表页与执行页的页面骨架规则，重点解决以下问题：

- 头部说明卡片过高，挤压首屏表格空间
- 主操作按钮与结果列表分离，用户视线来回跳转
- 同类页面骨架不一致，后续实现容易反复回退

适用页面：

- `art-design-pro/src/views/asset/use/index.vue`
- `art-design-pro/src/views/asset/disposal/index.vue`
- `art-design-pro/src/views/asset/inventory/index.vue`
- `art-design-pro/src/views/asset/inventory/task/index.vue`

## 2. 核心规则

### 2.1 列表工作台页取消大头部卡片

- `asset/use`、`asset/disposal`、`asset/inventory` 这类列表工作台页，不再使用大面积说明型头部卡片。
- 页面顶部仅保留轻量页头，承载标题和一句摘要，不再单独占用大块卡片高度。
- 需要说明业务边界时，优先放入表格卡片标题下的副标题，而不是再起一个说明卡片。

### 2.2 主操作统一进入表格卡片头工具栏

- 列表页的新增、发起、刷新、状态标签，应统一进入表格卡片头工具栏。
- 主操作按钮必须与结果列表处于同一视觉区域，避免出现“按钮在头部、数据在下方”的割裂体验。
- 典型布局如下：
  - `asset/use`：`发起领用 / 发起调拨 / 发起退还 / 刷新记录`
  - `asset/inventory`：`资产类型标签 / 刷新任务 / 发起盘点任务`
  - `asset/disposal`：`处置入口标签 / 刷新资产池`、`刷新记录`

### 2.3 执行页改为紧凑任务栏

- `asset/inventory/task` 这类执行页，不再使用大头部卡片和大摘要模块。
- 顶部采用紧凑任务栏，仅保留：
  - 返回入口
  - 页面标题
  - 任务号与状态标签
  - 计划盘点日、待登记数量、异常数、当前进度等紧凑标签
- 首屏空间优先让给“任务资产清单”，而不是让说明区和统计区占用表格高度。

### 2.4 搜索区与表格区仍保持分离

- 搜索栏仍应放在主表格卡片外部，避免与 `ArtTable` 共享高度计算容器。
- 但搜索栏必须保持紧凑，不再与头部说明卡片叠加形成多层纵向占高。

## 3. 判断标准

- 用户进入页面后，首屏首先看到的是“可操作的数据区域”，而不是装饰性信息块。
- 如果某个按钮的作用是“对当前列表做新增、发起、批量处理、刷新”，它默认应该在当前表格卡片头工具栏。
- 如果某段信息不能直接帮助当前页完成动作，它应被收敛为副标题、标签或折叠提示，而不是独立卡片。

## 4. 落地建议

建议后续资产模块页面实现按下面顺序统一：

1. 先统一列表工作台页：`use/index`、`disposal/index`、`inventory/index`
2. 再统一执行页：`inventory/task`
3. 最后统一流程页和详情页，避免不同页面继续混用旧骨架

## 5. 表格不遮挡的强制约束（新增）

针对本轮暴露的“分页存在但底部被裁切”问题，后续资产页必须遵守以下约束：

1. 卡片高度链必须完整：
  - 卡片容器：`display: flex; flex-direction: column; min-height: 0;`
  - 卡片 body：`flex: 1; min-height: 0; height: auto;`
2. 资产工作台页面禁止让 `el-card__body` 使用固定 `height: 100%`。
3. 资产工作台页面统一覆盖表格默认固定高度行为，避免分页落入裁切区：
  - `:deep(.art-table .el-table) { height: auto !important; margin-top: 0; }`
4. 表格容器仍使用 `flex: 1; min-height: 0;`，保证首屏可见数据区最大化。

建议作为资产模块页面模板片段复用，避免后续新增页面再次出现同类问题。
