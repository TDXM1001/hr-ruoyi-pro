# 2026-03-19 资产模块前端体验设计补充

## 1. 目标与适用范围

本文档用于补充资产模块前端页面的统一体验规范，解决“说明区过厚、首屏表格可视空间过小、操作层级过深、页面骨架不统一”的问题。

适用范围：

- `art-design-pro/src/views/asset/ledger/`
- `art-design-pro/src/views/asset/inventory/`
- `art-design-pro/src/views/asset/use/`
- `art-design-pro/src/views/asset/disposal/`

本文档作为资产模块后续页面实现、重构和验收时的统一参考，不替代业务流程设计，而是约束前端呈现方式。

## 2. 设计目标

- 首屏优先展示“可操作内容”，而不是堆叠说明区和装饰性卡片。
- 列表页以“工作台”视角设计，优先保证筛选、批量操作、表格浏览和分页切换效率。
- 流程页以“任务完成”视角设计，优先保证步骤清晰、状态明确、底部操作稳定可见。
- 详情页以“查阅和追踪”视角设计，优先保证信息分组清晰，避免一屏堆叠过多描述卡片。
- 同类页面采用同一骨架，降低用户学习成本和后续维护成本。

## 3. 页面分型与推荐骨架

### 3.1 工作台型列表页

适用页面：

- 资产台账列表
- 资产盘点任务中心
- 资产使用交接记录
- 资产处置列表

推荐骨架：

1. 紧凑页头
2. 筛选栏
3. 主表格工作区

设计规则：

- 页头只保留返回、标题、关键状态标签、主操作按钮，不再放长段说明。
- 说明信息如有必要，使用一行摘要或折叠提示，不长期占用首屏高度。
- 筛选栏与主表格分离，不能与 `ArtTable` 共用一个需要自行计算高度的容器。
- 主表格卡片应成为页面视觉中心，批量操作、刷新、统计标签放在表格头部工具条中。
- 不允许在列表页主表格上方叠两层以上独立信息卡片。

### 3.2 强操作任务页

适用页面：

- 盘点执行页 `asset/inventory/task`
- 后续若有批量审批、批量核查等任务页，也应沿用本型

推荐骨架：

1. 紧凑页头
2. 任务状态条
3. 筛选栏
4. 主表格工作区
5. 右侧抽屉或侧边登记区

设计规则：

- 页头只展示任务名称、任务号、状态、返回入口。
- 进度、异常数、当前页数、已选数统一收敛到任务状态条或表格工具条中。
- 批量登记按钮必须靠近“已选数量”展示，形成明确动作闭环。
- 异常登记优先使用抽屉，不使用全局弹窗反复打断列表浏览。
- 首屏优先展示资产清单，不允许让说明、进度、搜索、卡片标题共同挤压表格区。

### 3.3 流程表单页

适用页面：

- 资产台账新增/编辑
- 资产交接单新增
- 盘点任务发起页

推荐骨架：

1. 页头摘要
2. 单一纵向滚动内容区
3. 固定底部操作条

设计规则：

- 表单内容与资产选择区域使用同一个主滚动容器，避免多重滚动冲突。
- 页面底部操作条固定，始终保持“取消 / 上一步 / 下一步 / 提交”等动作可见。
- 规则说明、填报提示、字段解释优先折叠或收敛到轻提示中。
- 表单按业务分组，但分组数量应受控，避免单页出现过多 section。

### 3.4 详情查阅页

适用页面：

- 资产台账详情
- 交接单详情

推荐骨架：

1. 顶部摘要卡
2. Tab 分区内容

设计规则：

- 页头摘要只保留最关键识别信息和主要状态。
- 详细信息通过 Tab 或明确分区展开，不建议一屏堆叠过多说明卡片。
- 详情页优先支持“快速定位信息”，而不是“从上到下阅读整页”。

## 4. 统一布局规则

### 4.1 页头规则

- 列表页页头高度建议控制在 `72px ~ 96px`。
- 流程页页头允许适度增加，但不建议超过 `140px`。
- 长说明默认不常驻展开，最多展示一行摘要。

### 4.2 筛选栏规则

- 默认只展示高频筛选项，低频筛选项收纳到“展开”区域。
- 筛选栏必须位于表格卡片外部，避免参与 `ArtTable` 高度计算。
- 同一页默认首行展示 3 到 4 个核心筛选项即可，不追求“一次全铺满”。

### 4.3 表格工作区规则

- 列表页的主表格必须占据页面剩余可视空间。
- 批量操作、已选数量、刷新动作统一收敛到表格工具条，不另起一块大卡片。
- 表格卡片应尽量减少嵌套，不再在表格外再包一层非必要的说明卡或统计卡。

### 4.4 操作区规则

- 主操作按钮只保留一个强调动作，其余为次级按钮或文字按钮。
- 批量操作应依赖“已选数量”进行启用/禁用反馈，避免用户无感点击。
- 重要状态如任务正常、任务逾期、可选状态范围，应使用标签表达，而不是长文本堆叠。

## 5. 可视空间与密度基线

为避免“页面能看不能用”，资产模块页面应满足以下最低基线：

- 在 `1920 x 1080` 分辨率下，列表页首屏至少可见 `8 ~ 12` 行表格数据。
- 在 `1366 x 768` 分辨率下，列表页首屏至少可见 `5 ~ 6` 行表格数据。
- 表格分页区必须完整可见，不能被卡片底部或外层容器裁剪。
- 空态图、分页器、批量工具条不能依赖用户先滚动到看不见的位置才能发现。

## 6. 统一交互原则

- 用户做高频任务时，页面优先服务于“筛选、勾选、处理、提交”，而不是展示背景信息。
- 用户做流程录入时，页面优先服务于“当前步骤、当前状态、当前动作”，而不是一次展示全部说明。
- 用户做详情查阅时，页面优先服务于“快速定位字段、查看留痕、进入相关流程”，而不是静态排版美观。
- 同一业务动作的入口位置应稳定，例如批量按钮始终位于表格工具条右侧。

## 7. 明确禁止的反模式

- 在列表页中同时堆叠“页头卡 + 说明卡 + 统计卡 + 搜索卡 + 表格卡”。
- 把 `ArtSearchBar`、统计信息和 `ArtTable` 放进同一个需要自适应计算高度的容器中。
- 为了展示说明而牺牲表格首屏行数，导致页面进入后“看得到标题，看不到数据”。
- 在任务执行页把进度、规则、批量操作、搜索栏分别拆成多块卡片，形成首屏空间碎片化。
- 在流程页使用多个嵌套滚动容器，导致用户难以判断滚动区域。

## 8. 后续实现建议

- 资产模块后续页面实现优先按“页面分型”套骨架，而不是每页重新设计。
- 先统一列表页和任务页骨架，再优化表单页和详情页。
- 若新增新页面，应先判定其属于“工作台型列表页 / 强操作任务页 / 流程表单页 / 详情查阅页”中的哪一种，再开始实现。

## 9. 落地优先级

建议后续按下面顺序治理资产模块前端体验：

1. 列表工作台页：`ledger/index`、`inventory/index`、`use/index`、`disposal/index`
2. 强操作任务页：`inventory/task`
3. 流程页：`inventory/create`、`use/form`、`ledger/form`
4. 详情页：`ledger/detail`、`use/detail`

## 10. 作为验收参考的最小检查清单

- 页面进入后，用户能在首屏直接找到主操作按钮。
- 页面进入后，用户能在首屏直接看到主数据表格或主表单内容。
- 页面进入后，不需要先滚动才能发现分页器或底部操作区。
- 批量操作页中，用户能明确知道当前已选数量与下一步动作。
- 同类资产页面骨架一致，用户无需重新学习页面布局。
