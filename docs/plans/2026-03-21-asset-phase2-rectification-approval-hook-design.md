# 2026-03-21 不动产整改审批挂载位第一批设计

## 1. 背景

当前分支已经完成以下闭环：

1. 不动产档案建账闭环
2. 巡检异常到整改完成闭环
3. 整改完成后的只读收口与变更留痕

但整改线仍停留在“业务完成即收口”的状态，尚未挂接审批主线。结合当前代码和工作区状态，本批不直接接入通用审批基础设施，而是在不动产整改域内先做一套独立、可演示、可回归的审批挂载位最小闭环。

## 2. 本批目标

本批目标不是做完整审批平台，而是完成以下最小业务闭环：

> 整改单完成 -> 提交审批 -> 审批通过或驳回 -> 结果回写整改单 -> 详情页可见审批状态与轨迹

## 3. 约束

1. 不触碰当前工作区中已有的审批/处置脏文件
2. 不改动 `application-druid.yml`
3. 不引入新的通用审批中心页面
4. 继续复用不动产详情壳、整改页签、整改单编辑页、整改完成页

## 4. 设计范围

### 4.1 数据层

本批新增整改审批专用字段和轨迹表，不复用通用审批记录表。

新增到整改单主表的字段：

1. `approval_status`：审批状态
2. `approval_submitted_time`：提交审批时间
3. `approval_finished_time`：审批完成时间

新增整改审批轨迹表，用于记录：

1. 提交审批
2. 审批通过
3. 审批驳回

## 5. 状态设计

### 5.1 整改单状态

整改单主状态维持现有口径：

1. `PENDING`
2. `COMPLETED`

### 5.2 审批状态

新增审批挂载状态：

1. `UNSUBMITTED`
2. `SUBMITTED`
3. `APPROVED`
4. `REJECTED`

### 5.3 状态规则

1. 只有 `COMPLETED` 的整改单才能提交审批
2. `UNSUBMITTED` 和 `REJECTED` 可以提交审批
3. 只有 `SUBMITTED` 状态才能执行“审批通过 / 审批驳回”
4. `APPROVED` 后不允许再次提交审批
5. `REJECTED` 后允许再次提交审批
6. 本批不改动整改单主状态，审批结果作为挂载状态独立展示

## 6. 页面设计

### 6.1 整改页签

在整改卡片中新增审批信息区：

1. 审批状态标签
2. 提交时间
3. 审批完成时间
4. 最新审批意见摘要

新增动作：

1. `提交审批`
2. `审批通过`
3. `审批驳回`
3. `查看审批轨迹`

说明：

- 当前批次为了闭环点测，审批动作先挂在整改页签卡片内
- 不单独创建审批中心页面
- 后续若接入正式审批中心，再将动作迁出

### 6.2 整改完成页

新增审批挂载信息卡片：

1. 当前审批状态
2. 审批轨迹摘要
3. 若已完成但未提交审批，展示 `提交审批` 入口

### 6.3 整改单编辑页

已完成场景下补充审批只读信息区：

1. 审批状态
2. 提交时间
3. 审批完成时间
4. 最近审批意见

## 7. 接口设计

在不动产整改接口下新增：

1. `POST /asset/real-estate/{assetId}/rectifications/{rectificationId}/submit-approval`
2. `POST /asset/real-estate/{assetId}/rectifications/{rectificationId}/approve`
3. `POST /asset/real-estate/{assetId}/rectifications/{rectificationId}/reject`
4. `GET /asset/real-estate/{assetId}/rectifications/{rectificationId}/approval-records`

## 8. 权限策略

本批不新增独立审批权限种子，暂复用现有：

1. 查询动作复用 `asset:realEstate:query`
2. 审批提交与审批动作复用 `asset:realEstate:edit`

这是本批为避免碰撞现有脏文件采取的收敛策略，后续接正式审批主线时再拆权限。

## 9. 验证口径

本批完成后，至少要证明以下链路成立：

1. 已完成整改单可提交审批
2. 提交后审批状态变为 `SUBMITTED`
3. 已提交整改单可审批通过
4. 已提交整改单可审批驳回
5. 驳回后允许再次提交审批
6. 详情页整改卡片能看到审批状态变化
7. 审批轨迹可查询并展示

## 10. 本批边界

本批不做：

1. 通用审批中心
2. 多节点审批流
3. 处置审批复用
4. 审批通过后的资产终态统一回写
5. 与财务/采购/合同集成

本批只做“整改审批挂载位第一批”，目标是接通下一条闭环线，而不是一次做完整审批体系。
