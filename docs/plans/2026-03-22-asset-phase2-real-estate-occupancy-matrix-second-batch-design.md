# 2026-03-22 不动产占用管理状态矩阵第二批设计

## 1. 背景

当前不动产占用页签已经具备发起占用、变更占用、释放占用和历史记录展示能力，但对资产管理员而言仍有三个缺口：

1. 缺少明确的状态矩阵提示，用户需要自己判断当前可执行动作。
2. 变更原因与释放原因完全手填，重复输入较多，口径不一致。
3. 占用轨迹缺少筛选能力，记录增多后无法快速定位目标记录。

本批只做前端增强，不扩展后端模型，不引入新的后端字典或配置中心。

## 2. 目标

在不动产详情页的占用页签内补齐第二批体验能力：

1. 展示“占用状态矩阵”，说明不同占用状态下的可执行动作。
2. 为发起/变更原因与释放原因提供模板快捷填充。
3. 为占用历史提供状态筛选和关键字检索。

## 3. 范围

### 3.1 本批包含

1. 占用页签右侧提示卡升级为状态矩阵卡。
2. 占用历史记录区增加筛选工具条。
3. 发起/变更占用 Drawer 增加原因模板。
4. 释放占用 Drawer 增加释放原因模板。
5. 前端测试与浏览器点测更新。

### 3.2 本批不包含

1. 原因模板后端配置化。
2. 按日期区间、部门、责任人等高级筛选。
3. 占用状态新增更多状态枚举。

## 4. 交互设计

## 4.1 状态矩阵

占用页签展示三条状态规则：

1. `无有效占用`
   - 可执行：发起占用
   - 不可执行：变更占用、释放占用
2. `存在有效占用`
   - 可执行：变更占用、释放占用
   - 结果：历史轨迹保留，资产主档同步回写
3. `已释放历史`
   - 可执行：仅查看轨迹
   - 不可执行：直接对历史单再次变更或释放

状态矩阵只表达当前业务规则，不承担流程入口本身。真正动作仍从当前占用卡片按钮或空态入口发起。

## 4.2 原因模板

### 发起/变更原因模板

使用同一组模板：

1. 部门调整交接
2. 办公区域调整
3. 项目入驻占用
4. 台账口径校正

点击模板后，直接写入原因文本框，用户仍可继续编辑。

### 释放原因模板

1. 部门搬离释放
2. 装修改造临时释放
3. 处置前清场
4. 权属收回

同样采用“点击即填充，可继续编辑”的规则。

## 4.3 轨迹筛选

历史区增加两类筛选：

1. 状态筛选：`全部 / 有效占用 / 已释放`
2. 关键字检索：匹配占用单号、使用部门、责任人、位置、原因文本

筛选只作用于历史区，不影响当前有效占用卡片。

## 5. 技术方案

1. `occupancy-panel.vue`
   - 新增状态矩阵区
   - 新增 `statusFilter` 和 `keyword` 本地状态
   - 新增 `filteredRecords` 计算属性
2. `detail/index.vue`
   - 新增原因模板常量
   - 新增模板点击填充方法
   - 在 Drawer 的原因输入项下渲染模板按钮组
3. 测试
   - 占用页签组件测试覆盖状态矩阵与轨迹筛选
   - 详情页测试覆盖变更原因模板和释放原因模板

## 6. 验证口径

1. 占用页签能看到状态矩阵说明。
2. 点击状态筛选后，历史区只保留对应状态记录。
3. 输入关键字后，历史区能准确过滤记录。
4. 点击变更原因模板，变更 Drawer 原因文本框被填充。
5. 点击释放原因模板，释放 Drawer 原因文本框被填充。
