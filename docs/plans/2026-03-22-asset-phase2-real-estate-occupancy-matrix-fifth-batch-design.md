# 不动产占用状态矩阵第五批设计

## 1. 目标

在现有占用页签基础上，继续补足三个直接影响资产管理员效率的能力：

1. 摘要卡片点击联动轨迹定位
2. 筛选条件持久化
3. 导出当前占用轨迹

本批仍限定为前端体验增强，不新增后端接口，不变更数据库结构。

## 2. 设计边界

### 2.1 保持不变

- 不新增 SQL
- 不修改占用领域接口
- 不新增独立页面
- 不改变当前占用、变更占用、释放占用的业务口径

### 2.2 本批新增

- 允许从摘要卡片快速跳到对应轨迹
- 允许按资产维度恢复用户上一次筛选状态
- 允许将当前筛选结果导出为 CSV 文件

## 3. 交互设计

### 3.1 摘要卡片点击联动轨迹定位

#### 主档联动摘要

- 点击“主档联动摘要”或其中差异项时：
  - 自动切到 `有效占用`
  - 自动清空关键字
  - 自动重置时间范围为 `全部时间`
  - 保持轨迹排序为 `最新在前`
  - 滚动到当前有效占用轨迹
  - 对目标轨迹做短暂高亮

#### 最近一次变更摘要

- 点击“最近一次变更摘要”或其中差异项时：
  - 自动切到 `已释放`
  - 自动清空关键字
  - 自动重置时间范围为 `全部时间`
  - 保持轨迹排序为 `最新在前`
  - 滚动到最近一条已释放轨迹
  - 对目标轨迹做短暂高亮

### 3.2 筛选条件持久化

- 持久化维度：
  - 状态筛选
  - 时间筛选
  - 自定义起止日期
  - 排序方向
  - 关键字
- 存储介质：`localStorage`
- 存储键：`asset-real-estate-occupancy-filters:<assetCode>`
- 恢复时机：
  - 组件挂载时恢复
  - 资产编码变化时重新按新 key 读取

### 3.3 导出当前占用轨迹

- 导出范围：当前 `filteredRecords`
- 导出格式：`CSV`
- 导出列：
  - 占用单号
  - 占用状态
  - 使用部门
  - 责任人
  - 使用位置
  - 占用起始
  - 释放时间
  - 发起/变更原因
  - 释放原因
- 文件名：`<assetCode>-occupancy-history.csv`

## 4. 组件改造

目标组件：

- `art-design-pro/src/views/asset/real-estate/detail/components/occupancy-panel.vue`

新增内容：

- 摘要卡片点击交互
- 轨迹项 `data-testid` 和聚焦样式
- 导出按钮

## 5. 测试设计

### 5.1 定向测试

新增测试场景：

1. 点击摘要卡片后联动到目标轨迹
2. 筛选条件在重新挂载后可恢复
3. 导出只导出当前筛选结果

### 5.2 回归测试

继续跑不动产详情页相关回归，避免影响：

- 占用页签现有筛选
- 巡检页签
- 整改页签
- 审批挂载位
