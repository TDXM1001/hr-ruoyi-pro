# 大型企业全功能 HR 系统设计方案 (第一期)

## 1. 概述
本项目旨在基于若依 RuoYi-Vue 后端与 art-design-pro 前端，构建一套适用于大型企业的全功能人事管理系统。方案采用“全量规划、分期实施”的策略，第一期重点完成组织架构、员工全生命周期档案及可视化审批流程基建。

## 2. 核心架构设计
### 2.1 技术栈
- **后端**: Spring Boot 2.x (若依原生) + Flowable 6.x (嵌入式工作流引擎)
- **前端**: Vue 3 + Vite + Element Plus + Tailwind CSS
- **模块划分**:
  - `ruoyi-hr`: 独立业务模块，处理 HR 核心逻辑。
  - `ruoyi-workflow`: 工作流封装模块，处理 JSON ↔ BPMN 转换及 Flowable 集成。

### 2.2 数据域策略 (独立模式)
采用独立 HR 业务数据域，通过 `user_id` 与若依 `sys_user` 实现松耦合关联。
- 优点：支持复杂的 HR 业务模型（如多维组织架构、职级体系），不破坏若依原生权限体系，方便后续升级。

## 3. 数据库模型 (第一期)
### 3.1 组织与岗位
- `hr_organization`: 支持集团、公司、部门等 6 级以上架构及树形管理。
- `hr_position_category`: 职族管理（技术、管理、职能等）。
- `hr_position_level`: 职级体系（P1-P10, M1-M6）。
- `hr_position`: 岗位编制管理。

### 3.2 员工与合同
- `hr_employee`: 员工核心档案（入职、转正、离职状态流转）。
- `hr_employee_education`: 教育背景。
- `hr_employee_work_history`: 职业经历。
- `hr_contract`: 劳动合同生命周期管理（含到期预警）。
- `hr_employee_change`: 异动记录（入、转、调、离）。

## 4. 可视化审批流设计
### 4.1 可视化编辑器
采用钉钉风格的纵向树状流程设计器：
- **节点类型**: 发起人、审批人（直属上级/指定人/角色/部门负责人）、抄送人、条件分支、结束。
- **配置**: 支持或签、会签、条件表达式配置。

### 4.3 业务集成
- **预设模板**: 入职审批、转正申请、调岗申请、离职流程。
- **状态联动**: 审批通过后自动触发 HR 业务数据状态变更（如：转正审批通过 -> 修改员工状态为“正式”）。

## 5. 前端规划
- **员工花名册**: 提供多维过滤、360度档案视图、批量导入导出。
- **组织架构图**: 提供可视化组织树，支持拖拽调整。
- **审批中心**: 统一的任务处理入口（待办、已办、我发起）。

## 6. 实施路线
- **Phase 1 (当前)**: 核心档案 + 组织管理 + 审批流基建。
- **Phase 2**: 考勤假勤管理（排班、打卡、假额）。
- **Phase 3**: 薪酬绩效管理（薪资核算、KPI/OKR）。
- **Phase 4**: 招聘、培训与人才发展。
