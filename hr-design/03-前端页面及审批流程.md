# 前端页面及审批流程设计

## 一、 前端菜单与路由结构（第一期）

```text
人事管理 (hr)                              ← 一级菜单
├── 组织管理 (hr/organization)
│   ├── 组织架构 (hr/organization/tree)     ← 树形可视化 + 列表双视图
│   └── 组织变更 (hr/organization/change)   ← 变更记录列表
├── 岗位职级 (hr/position)
│   ├── 职族管理 (hr/position/category)
│   ├── 职级管理 (hr/position/level)
│   └── 岗位管理 (hr/position/list)         ← 含编制/在岗人数统计
├── 员工管理 (hr/employee)
│   ├── 员工花名册 (hr/employee/list)       ← 主列表页
│   ├── 入职管理 (hr/employee/onboard)      ← 待入职 / 办理入职
│   ├── 转正管理 (hr/employee/regular)      ← 待转正列表
│   ├── 调动管理 (hr/employee/transfer)     ← 调岗/晋升记录
│   └── 离职管理 (hr/employee/leave)        ← 离职办理
├── 合同管理 (hr/contract)
│   ├── 合同列表 (hr/contract/list)
│   └── 合同到期提醒 (hr/contract/expiring) ← 自动提醒即将到期
└── 员工档案详情 (hr/employee/detail/:id)    ← 隐藏路由，员工360视图

审批中心 (workflow)                          ← 一级菜单
├── 我的待办 (workflow/todo)                 ← 待我审批的任务
├── 我的已办 (workflow/done)                 ← 我已处理的任务
├── 我发起的 (workflow/mine)                 ← 我提交的申请
└── 流程管理 (workflow/process)              ← 管理员可见
    ├── 流程设计 (workflow/process/design)   ← 钉钉风格可视化设计器
    └── 流程部署 (workflow/process/deploy)   ← 流程定义列表
```

## 三、 审批流程设计（第一期 · 精简版）

**设计模式：可视化流程设计器 + JSON 互转 BPMN 引擎支持**。第一期侧重引入钉钉风格的流程图与预设表干功能。

### 1. 可视化流程设计器 — 节点类型

| 节点类型 | 图标颜色 | 功能 |
|---------|---------|------|
| **发起人** | 🔵 蓝色 | 流程起始节点，自动设置 |
| **审批人** | 🟠 橙色 | 指定审批人（固定人员/直属上级/部门负责人/角色） |
| **抄送人** | 🟢 绿色 | 通知节点，不需要操作 |
| **条件分支** | 🟡 黄色 | 互斥条件（如：请假天数>3天走部门总监审批） |
| **结束** | ⚫ 灰色 | 流程结束节点 |

### 2. 预设流程模板

| 模板名称 | 默认流程 |
|---------|---------|
| **入职审批** | 发起人 → HR 专员审核 → 部门负责人确认 → HR 经理批准 → 结束 |
| **转正审批** | 发起人(员工) → 直属上级评估 → 部门负责人 → HR 审核 → 结束 |
| **调岗审批** | 发起人(HR) → 原部门负责人 → 新部门负责人 → HR 经理 → 结束 |
| **离职审批** | 发起人(员工) → 直属上级 → 部门负责人 → HR → IT(回收权限) → 结束 |
| **合同续签** | 发起人(HR) → 部门负责人意见 → HR 经理审核 → 结束 |

### 3. 审批人指定策略

```text
审批人节点配置项:
├── 指定人员     → 手动选择具体用户
├── 直属上级     → 自动取 hr_employee.direct_leader_id
├── 部门负责人   → 自动取 hr_organization.leader_id
├── 角色        → 取拥有指定角色的用户（如"HR经理"角色）
├── 发起人自选   → 发起时由申请人选择审批人
└── 审批方式
    ├── 或签（任一人通过即可）
    └── 会签（所有人都需通过）
```
